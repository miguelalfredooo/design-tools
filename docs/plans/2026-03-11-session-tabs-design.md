# Session Tabs Design

Add three tabs to the session detail page:

- `PRD / Brief`
- `Design Plan`
- `Research`

## Default

`Design Plan` is the default selected tab.

## Mapping

### PRD / Brief

- session brief
- evidence status / validation

### Design Plan

- participant controls
- voting progress
- results
- voting instructions
- option grid

### Research

- session insights
- synthesize action / empty research state

## Why

- separates framing from design work
- keeps research/evidence from competing with voting UI
- makes the session page easier to scan and less vertically overloaded
